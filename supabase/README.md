# مخطط قاعدة البيانات — منصة أمانة (Supabase)

هذا المجلد يحتوي على ملفات ترحيل قاعدة البيانات (SQL migrations) الخاصة بمنصة أمانة على Supabase Postgres. الملفات **لم تُطبَّق بعد** — هذا الدليل يشرح كيفية تطبيقها لاحقاً.

## الملفات

| الملف | الوصف |
|------|-------|
| `migrations/0001_init.sql` | المخطط الأساسي: الأنواع المعدودة (enums)، الجداول (`profiles`, `drivers`, `rides`, `ratings`, `groups`, `group_members`)، الدوال والمشغّلات، الفهارس، وتفعيل RLS. |
| `migrations/0002_rls.sql` | سياسات Row Level Security لكل جدول، معتمدة على `auth.uid()`. |

ترتيب التطبيق مهم: يجب تطبيق `0001_init.sql` **قبل** `0002_rls.sql`.

## طريقة التطبيق (١): عبر Supabase CLI

إذا كان المشروع مربوطاً بـ Supabase عبر الـ CLI:

```bash
# ربط المجلد المحلي بمشروع Supabase البعيد (مرة واحدة)
supabase link --project-ref <your-project-ref>

# دفع كل الترحيلات الموجودة في supabase/migrations إلى القاعدة البعيدة
supabase db push
```

يقرأ الأمر `supabase db push` الملفات من `supabase/migrations` بترتيب أسمائها (`0001` ثم `0002`) ويطبّقها.

للتجربة محلياً أولاً على قاعدة Supabase المحلية:

```bash
supabase start
supabase db reset   # يعيد بناء القاعدة المحلية ويطبّق كل الترحيلات
```

## طريقة التطبيق (٢): عبر محرّر SQL في لوحة Supabase

بدون الـ CLI، يمكن لصق المحتوى يدوياً:

1. افتح لوحة تحكم المشروع على `app.supabase.com`.
2. من القائمة الجانبية اختر **SQL Editor** ثم **New query**.
3. الصق كامل محتوى `migrations/0001_init.sql` واضغط **Run**.
4. أنشئ استعلاماً جديداً، الصق كامل محتوى `migrations/0002_rls.sql` واضغط **Run**.

احرص على تشغيل `0001` قبل `0002`.

## ملاحظات مهمة

- **دور المشرفة (admin):** لوحة المشرفة تستخدم مفتاح `service_role` الذي **يتجاوز RLS** بالكامل، لذلك لا توجد سياسات RLS خاصة بالمشرفة. لا تُعرّض مفتاح `service_role` أبداً في تطبيق العميل — يُستخدم فقط في الخادم.
- **إنشاء الملفات الشخصية:** يُنشأ سطر في جدول `profiles` تلقائياً لكل مستخدم جديد عبر المشغّل `on_auth_user_created` على `auth.users`، لذلك لا حاجة (ولا سياسة) للإدراج المباشر في `profiles` من العميل.
- **الدور والاسم عند التسجيل:** تُقرأ `role` و`full_name` من `raw_user_meta_data` عند التسجيل؛ مرّرهما ضمن `options.data` في استدعاء `signUp`.
- **قابلية إعادة التشغيل:** الملفات مكتوبة لتكون آمنة عند إعادة التشغيل قدر الإمكان (`create ... if not exists`, `drop policy if exists ... create policy`)، لكن يُفضَّل تطبيقها مرة واحدة بالترتيب.
