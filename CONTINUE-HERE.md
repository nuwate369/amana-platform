## ✅ ما أُنجز (الجلسة الحالية)

**1. فئة المركبة (Vehicle Class):**
- الراكب: يختار الفئة وتُحفظ فعلًا في الرحلة (`rides.requested_class`).
- السائق: يُدخل الفئة في التوثيق KYC (standard/premium/group)، وتُحفظ في `drivers.vehicle_class`. 
- عرض الفئة المطلوبة للسائق على بطاقة الطلب الجديد.

**2. إشعارات لوحة الإدارة (`announcements`):**
- بناء جدول للإشعارات الحقيقية (`0037` و `0038`).
- صفحة إشعارات متكاملة (فلاتر، إحصاءات، نموذج إرسال لـ الكل/الركاب/السائقات/مستخدم محدد).
- **جديد:** منطق تاريخ الانتهاء `expires_at` بحيث لا يظهر الإشعار للمستخدمين الجدد بعد انتهائه.

**3. الدعم الذكي للراكب (Groq AI):**
- بناء نظام دعم ذكي للراكب (محادثات + تصعيد لتذاكر حقيقية) بنفس بنية السائق.

**4. تحسينات الخريطة والتتبع:**
- أيقونة السائقة سيارة بدل نقطة، مع رسم مسار (Route) كحلي/أبيض بين نقطة الالتقاط والوجهة (عبر Mapbox Directions).
- إصلاحات UI الراكب: هامش safe-area في التتبع، زر الإشعارات فوق الخريطة، إخفاء أزرار الشريط السفلي (إشعارات/حسابي) لمنع التكرار، ودعم لوحة المفاتيح (KeyboardAvoidingView) في تسجيل الدخول.

**7. إخفاء أيقونات الشريط السفلي المكررة (الراكب):**
- تبويبا «الإشعارات» و«حسابي» في `app/(tabs)/_layout.tsx` محدَّثان بـ `href: null` — لا يظهران في الشريط السفلي.
- الأيقونتان موجودتان في الشريط العلوي للرئيسية فقط، لا تكرار.
- **للرؤية:** شغّل dev-client محلياً (بدون بناء جديد): `npx expo start --dev-client -c --port 8082` ثم اتصل من الهاتف عبر WiFi.

**5. الميزات الإضافية للرحلة:**
- ميزة "طلب بدون وجهة": يمكن للراكبة تخطي الوجهة (وتُسجل null)، والسائقة ترى "بدون وجهة محددة".
- تقييم السائق للراكب: عند إنهاء الرحلة، السائقة تقيّم الراكبة قبل العودة للرئيسية.

**6. الترجمة الإنجليزية:**
- تنظيف شامل للمفاتيح المكسورة ومراجعة الترجمة (100% تطابق مع العربي).

## 🚀 التشغيل المحلي (عبر WiFi — بدون بناء)
```bash
# طرفيتان منفصلتان (الهاتف على نفس Wi‑Fi الحاسوب)
npx expo start --dev-client -c --port 8081   # السائقة  → افتح dev-client على الهاتف → أدخل IP:8081
npx expo start --dev-client -c --port 8082   # الراكبة  → افتح dev-client على الهاتف → أدخل IP:8082
npm run admin                                # http://localhost:3002 (متصفّح)
```

### خطوات الاتصال من الهاتف:
1. تأكد أن الهاتف والحاسوب على **نفس الشبكة (WiFi)**.
2. في تطبيق الـ dev-client على الهاتف → اضغط **Enter URL manually**.
3. أدخل: `http://<IP الحاسوب>:8082` (IP يظهر في طرفية Expo بعد `Metro waiting on`).
4. أو امسح **QR Code** الذي يظهر في الطرفية مباشرة.

- القاعدة الذهبية: **السائقة = 8081، الراكبة = 8082 دائمًا**.
- إن فشل الاتصال رغم Wi‑Fi: اسمح لـ Node.js عبر جدار حماية Windows (Private networks).
- لإيقاف الخادم: `Ctrl+C` في الطرفية.
- فحص الأنواع: `npm run typecheck` من الجذر.
