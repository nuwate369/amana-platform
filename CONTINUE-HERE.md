## ✅ ما أُنجز (هذه الجلسة)

**دورة الرحلة الكاملة (حقيقية، لا mock):**
- الراكب: طلب رحلة (خريطة + تسعير مسافة) → بحث Realtime → تتبّع حيّ → تقييم → **دفع تجريبي بأسعار حقيقية** (`payRide`/`computeInvoice`، هجرة 0033).
- **تدفّق تقوده السائقة:** قبول → «وصلت لنقطة الالتقاط» (عمود `driver_arrived_at`) → «بدء الرحلة» → «إنهاء الرحلة». الراكبة **تُشاهد فقط** وتنتقل تلقائيًّا للتقييم عند الإنهاء.
- السائقة: استقبال الطلبات + قبول (claim) + بثّ الموقع أثناء الرحلة.

**تفعيل المستخدم (هجرة 0034):**
- الراكب أيضًا يحتاج **تفعيل الإدارة** (نسخة التجربة). `profiles.status` (pending_approval→active) + عكس `email_confirmed_at` من auth.users.
- بوابة الراكب: شاشة `app/pending.tsx` تحبس غير المفعّل.
- اللوحة: شارة 4 حالات (نشط/بانتظار الموافقة/لم يؤكّد البريد/محظور) + أزرار تفعيل/إلغاء **مع حوار تأكيد** + `approvePassenger`/`revokePassenger`.
- تنبيهات الإدارة بالتسجيل الجديد (موحّدة، دفاعية).

**بيانات الراكب الحقيقية (كانت Stitch mock):**
- الرئيسية: خريطة حيّة + ترحيب بالاسم الحقيقي (أُزيلت البطاقات الوهمية).
- الملف: الاسم/البريد/تاريخ الانضمام + إحصاءات حقيقية (رحلات + تقييم).
- سجل الرحلات: رحلات فعلية من DB + تصفية. المنطق في `apps/passenger/src/lib/account.ts`.

**الشعار الجديد:** التصميم 2 (دبوس أخضر + امرأة بحجاب + سيارة). ولّدت نسخة iOS معتمة + نسخة أندرويد شفّافة بهامش أمان ٦٢٪ (تحلّ «الأيقونة كبيرة») في `assets/images/logo-amana*.png`، ورُبطت في `app.json`. **يحتاج بناءً**.

**سابقًا (مدفوع):** إصلاح تنبيه الجرس (قنوات Realtime فريدة + رفع elevation فوق Mapbox)، حلّ تشغيل التطبيقين (منافذ 8081/8082 + Wi‑Fi)، حلّ فشل تحميل الراكب (expo-dev-client).

## 🚀 التشغيل المحلي
```bash
# طرفيتان منفصلتان (الهاتف على نفس Wi‑Fi الحاسوب)
npm run driver      # Metro → 192.168.x.x:8081  (على الهاتف: أدخل http://<IP>:8081)
npm run passenger   # Metro → 192.168.x.x:8082  (على الهاتف: أدخل http://<IP>:8082)
npm run admin       # http://localhost:3002 (متصفّح)
```
- القاعدة الذهبية: **السائقة = 8081، الراكبة = 8082 دائمًا**.
- إن فشل الاتصال رغم Wi‑Fi: اسمح لـ Node عبر جدار حماية Windows (Private).
- فحص الأنواع: `npm run typecheck` من الجذر.

## 🧪 اختبار تدفّق التفعيل + الرحلة (بعد 0034 + 0035)
1. سجّل راكبة جديدة → أكّدي البريد → شاشة «قيد المراجعة».
2. اللوحة → الركاب → «تفعيل» (حوار تأكيد) → الراكبة «تحديث الحالة» → الرئيسية.
3. اطلبي رحلة → السائقة تقبل → «وصلت لنقطة الالتقاط» → «بدء» → «إنهاء» → الراكبة تنتقل للتقييم → دفع.
