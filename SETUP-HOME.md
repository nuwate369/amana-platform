# دليل تشغيل مشروع أمانة على جهاز جديد

> انسخ هذا الملف إلى الفلاش مع ملفات `.env` (لأنك ستحتاجه قبل استنساخ المشروع).

---

## الجزء الأول: قبل مغادرة الجهاز الحالي

انسخ ملفات `.env` إلى الفلاش — **هذه لا تُرفع إلى GitHub لأنها تحتوي أسرارًا**.
المطلوب **4 ملفات** (والخامس اختياري):

| # | الملف | مكانه الدقيق داخل المشروع |
|---|---|---|
| 1 | `.env` | جذر المشروع `amana-platform/` |
| 2 | `.env.local` | `amana-platform/apps/admin/` ← **الأهم (فيه المفتاح السرّي)** |
| 3 | `.env` | `amana-platform/apps/passenger/` |
| 4 | `.env` | `amana-platform/apps/driver/` |
| 5 | `.mcp.json` *(اختياري)* | جذر المشروع — لتصميم Stitch فقط، غير مطلوب للتشغيل |

> ملاحظة: هذه الملفات مخفية (تبدأ بنقطة). فعّل «إظهار الملفات المخفية» في مستكشف الملفات لرؤيتها ونسخها.

---

## الجزء الثاني: على جهاز المنزل

### 1) ثبّت المتطلبات أولًا
- **Node.js 20 أو أحدث** (نسخة LTS) — من https://nodejs.org
- **Git** — من https://git-scm.com

للتأكد من التثبيت، افتح Terminal/PowerShell ونفّذ:
```bash
node -v
git --version
```

### 2) استنسخ المشروع
افتح Terminal في المجلد الذي تريده (مثلًا `D:\`) ونفّذ:
```bash
git clone https://github.com/nuwate369/amana-platform.git
cd amana-platform
```
> إن طلب تسجيل دخول (المستودع خاص): أدخل حساب **nuwate369** أو Personal Access Token من GitHub.

### 3) ضع ملفات `.env` من الفلاش
انسخ الملفات الأربعة من الفلاش إلى **نفس مواقعها الدقيقة** (حسب جدول الجزء الأول).
بدونها لن يتصل التطبيق بقاعدة بيانات Supabase.

### 4) ثبّت الحزم
```bash
npm install
```
> الإعداد `legacy-peer-deps` مضبوط تلقائيًا في المشروع — لا تحتاج أي إضافة.

### 5) شغّل التطبيقات

| التطبيق | الأمر | يفتح على |
|---|---|---|
| لوحة الإدارة | `npm run admin` | http://localhost:3000 |
| تطبيق الراكبة | `npm run passenger` | Expo (امسح رمز QR بتطبيق Expo Go على جوالك) |
| تطبيق السائقة | `npm run driver` | Expo |

### 6) (اختياري) تحقّق أن كل شيء سليم
```bash
npm run typecheck
```

## تذكيرات مهمة

- **قاعدة البيانات مشتركة** بين جهازيك (نفس مشروع Supabase). أي بيانات أو مستخدمين تراهم على جهاز ستراهم على الآخر.
- لا يزال عليك **تطبيق هجرة `0007_user_type.sql`** (وربما `0010`–`0012`) في **Supabase SQL Editor** لتفعيل الأدوار كاملةً. تُطبَّق مرة واحدة فقط (لأن القاعدة مشتركة).
- **لا تضع ملفات `.env` في GitHub أبدًا** — هي مخصّصة للفلاش/النسخ اليدوي فقط.
- لسحب آخر تعديلات مستقبلًا: `git pull` داخل مجلد المشروع.
